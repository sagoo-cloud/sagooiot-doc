## 一、说明

在常规情况下，**设备是OPC UA服务端，平台是OPC UA客户端**。

### **1. OPC UA协议的设备：通常是服务端（Server）**
- **设备（PLC、传感器、仪器仪表等）作为数据源，天然承担服务端角色**
- 它们提供标准化的地址空间（Address Space），向外暴露数据节点、方法和事件
- 等待并响应客户端的连接请求，提供读、写、订阅等服务

### **2. 物联网平台系统：通常是客户端（Client）**
- **平台是数据的消费者和使用方，常规做法是作为客户端**
- 主动发起连接，从设备服务端读取数据
- 订阅数据变化、写入控制指令、调用设备方法
- 这种模式下，平台可以集中管理多个设备，符合"平台"的定位

---

### **常规架构示意图**
```
[设备层]                [平台层]                [应用层]
OPC UA Server  ←─────→  OPC UA Client  →  数据处理/可视化
(数据提供方)            (SagooIoT物联网平台)          (数据消费)
```

### **特殊情况的例外**
- **反向场景**：如果平台需要向其他系统（如MES、ERP）提供数据，可能同时充当**服务端**（OPC UA Server），但这不是它的"常规"角色
- **双向通信**：平台作为客户端是常规模式，服务端角色是扩展功能
